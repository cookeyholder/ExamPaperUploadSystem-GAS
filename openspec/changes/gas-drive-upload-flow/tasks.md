# Tasks: GAS Drive Upload Flow & Versioning

- [ ] Task 1: 撰寫 GAS 後端 Base64 轉 PDF 並上傳 Drive 邏輯。
  - **驗收標準**: 教師在前端 Modal 成功發送一個 5MB 的 PDF 後，管理者打開指定的 Google Drive 資料夾，能看到被正確命名的 PDF 實體檔案。
- [ ] Task 2: 實作重複上傳的「版本 / 覆蓋」處理。
  - **驗收標準**: 教師針對同一門科目重新上傳新的 PDF 後，Drive 中的舊檔案應被取代（或放入附屬資料夾），且該資料夾不會出現 `XXX (1).pdf`, `XXX (2).pdf` 這種亂象，始終只保留最新的有效檔案。
- [ ] Task 3: 寫入試算表與更新狀態。
  - **驗收標準**: 當檔案在 Drive 就緒後，GAS 會自動在該科目的「試卷檔案」欄位寫入 `HYPERLINK` 公式，並且在前端跳出「上傳成功」的提示。

- [ ] Task: 更新文件
  - **驗收標準**: 實作完成後，務必更新 `README.md` 的相關說明；如果涉及架構或業務邏輯的改變，請在 `docs/` 當中加入合適的文件。

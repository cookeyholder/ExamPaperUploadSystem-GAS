# Django 功能盤點與 GAS 尚待實作清單

> 說明：本文件以 Django 專案 `projects/Exam_Paper_Delivery_System` 的實際程式碼為準，盤點網站功能，並比對 `GAS-projects/ExamPaperUploadSystem-GAS` 目前尚待補齊項目與建議實作順序。

## 1. Django 網站功能總覽（完整清單）

### A. 首頁與導覽
- 首頁顯示「目前開放上傳中」的考試分項。
- 已登入教師可看到自己被指派的考科資訊。
- 站內路由包含：首頁、使用者管理、考試/科目管理、Django 後台、Captcha。

### B. 登入驗證與安全機制
- Email 登入流程。
- OTP 驗證流程（寄送一次性驗證碼、驗證成功後登入）。
- 圖形驗證碼（Captcha）與刷新驗證碼 API。
- OTP 嘗試次數限制（避免暴力嘗試）。
- 安全 `next` 轉址檢查（避免不安全跳轉）。
- 登出流程與快取清除。
- 登入成功通知信。

### C. 使用者管理（User）
- 使用者列表。
- 依科別篩選。
- 分頁與每頁筆數調整。
- 局部更新表格（partial/AJAX）。
- 新增、編輯、刪除使用者。
- Excel 匯入使用者（`.xlsx` + 檔案格式檢查）。

### D. 班級管理（Class）
- 班級列表。
- 依科別、年級篩選。
- 分頁與每頁筆數調整。
- 局部更新表格（partial/AJAX）。
- 新增、編輯、刪除班級。

### E. 科別管理（Department）
- 科別列表與分頁。
- 新增、編輯、刪除科別。

### F. 類群管理（Group of Department）
- 類群列表與分頁。
- 新增、編輯、刪除類群。

### G. 考試分項管理（ExamPlan）
- 考試分項列表（未封存）。
- 已封存考試分項列表。
- 新增、編輯、刪除考試分項。
- 複製考試分項（含複製 ESP 組合）。
- 封存 / 解除封存。
- 管理欄位：學年度、學期、考試名稱、調查起訖、上傳起訖。

### H. 科目管理（Subject）
- 科目列表。
- 依科別、年級、學年度、學期篩選。
- 分頁與局部更新表格。
- 新增、編輯、刪除、複製科目。
- Excel 匯入科目（`.xlsx` + 格式檢查 + 對應規則）。
- 刪除全部科目。
- 依學年度批次刪除科目。

### I. 考試分項科目組合管理（ESP）
- ESP 列表。
- 依科別、年級、上傳狀態篩選。
- 分頁與局部更新表格。
- 新增 ESP（可一次選多科目）。
- 刪除單筆 ESP。
- 批次刪除 ESP。
- 指派命題教師（含可指派教師規則）。

### J. 試卷上傳與檔案處理
- 上傳 PDF（檔案格式/大小檢查）。
- 設定閱卷方式。
- 填寫試卷張數。
- 英語文科目可填「是否含英聽加考」。
- 檔案雜湊值（SHA-256）。
- 舊檔覆蓋與舊檔清理。
- 上傳成功通知信。

### K. 檔案下載
- 單一試卷受保護下載（需通過權限檢查）。
- 整個考試分項打包 ZIP 下載（含清單說明）。

### L. 統計與報表
- 試卷張數統計頁（年級/科別/科目/班級/交叉統計）。
- 英語文英聽加考狀態統計。
- 匯出統計 Excel。

### M. 權限控管
- 大多功能需登入（`login_required`）。
- 各功能依權限代碼限制（`permission_required`）。
- 下載試卷含「物件層級」權限判斷。

## 2. GAS 專案尚待實作功能（目前缺口）

> 註：以下列的是「相對 Django 尚未補齊」的項目。GAS 目前已有 SPA 外框、部分 CRUD 畫面與基本讀寫 API。

### A. 認證與安全
- OTP 登入完整流程（含寄送、驗證、錯誤次數限制）。
- Captcha 驗證與刷新。
- 登出流程與安全轉址處理。
- 將 `verifyRole()` 真正套用到所有敏感寫入 API。
- 權限分級補強（管理員/教師操作邊界、禁止越權）。

### B. 使用者/班級/科別/類群管理補齊
- 使用者 Excel 匯入。
- 使用者列表篩選、分頁、局部更新。
- 班級資料補齊（科別、年級欄位與篩選）。
- 科別（Department）管理整組 CRUD。
- 類群（Group）管理整組 CRUD。

### C. 考試分項管理補齊
- 新增、刪除、複製考試分項。
- 封存/解除封存。
- 調查時間（investigate）欄位與驗證規則。

### D. 科目與 ESP 管理補齊
- 科目主檔完整欄位（學年度/學期/群組/課程類別/課程屬性/學分/時數/適用班級）。
- 科目列表完整篩選（學年度、學期、科別、年級）。
- 科目 Excel 匯入。
- 科目批次刪除（全部、依學年度）。
- ESP 的新增/刪除/批次刪除/篩選/分頁/局部更新。
- 命題教師可指派規則（一般科與專業科邏輯）。

### E. 試卷上傳與檔案
- `apiUploadExamPaper` 後端實作（目前前端有呼叫，GAS 後端尚未定義）。
- 檔案寫入 Google Drive、覆蓋舊檔、更新連結。
- 上傳檔案雜湊值（SHA-256）與欄位寫回。
- 英語文英聽加考欄位寫入與檢核。
- 上傳成功通知（Email）。

### F. 下載與統計
- 單一試卷受保護下載。
- 考試分項整包 ZIP 下載。
- 張數統計頁（各維度統計）。
- 統計 Excel 匯出。

### G. 系統一致性與資料模型
- GAS 試算表欄位與 Django 模型對齊（避免僅靠前端映射）。
- 部分頁面依賴 mock 資料欄位（如 `departments`）需改為正式來源。
- API 輸入/輸出驗證、錯誤訊息、例外處理一致化。

## 3. 建議實作先後順序（由高到低）

## 階段 0（先修，立即）
1. 補上 `apiUploadExamPaper` 後端函式，讓現有上傳流程可實際運作。
2. 所有寫入 API 套用權限檢查（至少先擋住非管理員寫入管理資料）。
3. 補齊資料表/欄位映射缺漏（特別是 `departments` 與 exam 資料欄位）。

## 階段 1（核心流程）
1. 考試分項：新增/刪除/複製 + 封存/解封存。
2. ESP：新增、刪除、批次刪除、指派教師規則。
3. 科目管理：完整欄位與篩選。

## 階段 2（管理效率）
1. 科目 Excel 匯入。
2. 使用者 Excel 匯入。
3. 班級/科別/類群管理補齊（含篩選與分頁）。

## 階段 3（安全強化）
1. OTP + Captcha 登入流程完整上線。
2. 登出、安全轉址、嘗試次數限制等安全細節。
3. 權限控管擴充到物件層級（例如只能下載自己負責的試卷）。

## 階段 4（報表與下載）
1. 單檔下載與整包 ZIP 下載。
2. 統計頁（張數、科別、年級、班級、英聽狀態）。
3. 統計 Excel 匯出。

## 階段 5（收斂與驗收）
1. 全流程測試（管理員、教師兩角色）。
2. 錯誤處理、訊息一致化、使用體驗修整。
3. 與 Django 功能逐項驗收簽核。

## 4. 建議驗收方式（簡要）
- 以本文件第 1 節功能為驗收清單，逐項標記：`未開始 / 開發中 / 完成 / 已驗證`。
- 每完成一個階段，至少做一次「管理員 + 教師」兩種角色的端對端測試。
- 先確保「能用且安全」，再做報表與優化。

# 部署前檢查清單（Checklist）

> 適用專案：`ExamPaperUploadSystem-GAS`
> 
> 使用方式：每次部署前逐項勾選，避免上線後才發現設定遺漏。

---

## 1. 開發環境檢查

- [ ] 已安裝 Node.js（建議 18+）
- [ ] 已安裝 npm
- [ ] 已安裝 `@google/clasp`
- [ ] 已完成 `clasp login`
- [ ] 專案根目錄 `npm install` 已成功

---

## 2. 原始碼與建置檢查

- [ ] `npm run build` 可成功完成
- [ ] 已確認 `dist/index.html` 有產出最新內容
- [ ] 已執行 `cp dist/index.html index.html`
- [ ] 已確認根目錄 `index.html` 是本次要部署的版本
- [ ] 本次修改內容已完成基本自我測試（至少手動點過主要頁面）

---

## 3. GAS 專案設定檢查

- [ ] `.clasp.json` 的 `scriptId` 已設定為正確目標專案
- [ ] `appsscript.json` 時區符合需求（目前為 `Asia/Taipei`）
- [ ] `Code.js` 已包含本次部署需要的後端函式
- [ ] 已執行 `clasp push` 並確認無錯誤

---

## 4. 試算表結構檢查

### 4.1 工作表存在
- [ ] `網站參數設定`
- [ ] `帳號管理`
- [ ] `群科班級`
- [ ] `第一次定期考`
- [ ] `第二次定期考`
- [ ] `第三次定期考`
- [ ] `期末考`

### 4.2 關鍵欄位（最少）

#### `網站參數設定`
- [ ] 有 `網站名稱`
- [ ] 有 `網站網域`
- [ ] 有 `OAuth Client ID`

#### `帳號管理`
- [ ] 欄位包含：`Email`, `姓名`, `人員編號`, `部門單位`, `群組`, `狀態`, `備註`
- [ ] 至少有 1 個可登入的管理員帳號（`admin`）

#### `群科班級`
- [ ] 欄位包含：`班級代碼`, `班級名稱`

#### `exam` 相關工作表（第一次~期末）
- [ ] 欄位包含：`ID`, `類別`, `年級`, `科目名稱`, `命題教師`, `閱卷方式`, `試卷張數`, `檔案連結`

---

## 5. OAuth 與存取設定檢查

- [ ] Google Cloud 已建立 OAuth Client ID（Web 應用程式）
- [ ] `OAuth Client ID` 已填入 `網站參數設定`
- [ ] OAuth 設定中的 JavaScript 來源與重新導向 URI 已正確設定
- [ ] Web App 部署時「執行身分」與「存取對象」符合預期

---

## 6. 功能驗收（部署後立即確認）

### 6.1 共通
- [ ] 開啟 Web App 不會白畫面或報錯
- [ ] 側欄與路由可正常切換
- [ ] 目前登入帳號名稱可正常顯示

### 6.2 管理員路徑
- [ ] 可開啟「考試分項管理」
- [ ] 可開啟「科目管理」
- [ ] 可開啟「群科班級管理」
- [ ] 可開啟「帳號管理」
- [ ] 至少完成一次新增/編輯/刪除（依本次變更範圍）

### 6.3 教師路徑
- [ ] 可看到教師儀表板
- [ ] 可看到被指派科目（若有資料）
- [ ] 可正常開啟上傳視窗

---

## 7. 已知限制提醒（目前版本）

- [ ] 已確認本次需求不依賴 `apiUploadExamPaper` 後端（此函式目前尚未完成）
- [ ] 已確認本次需求不依賴 Django 尚未遷移功能（OTP/Captcha、統計匯出、封存流程等）

---

## 8. 回滾與紀錄

- [ ] 已保留本次部署前版本（Git commit / tag）
- [ ] 已記錄本次部署時間、部署人、變更摘要
- [ ] 已記錄若需回滾時要恢復的版本資訊
